dir-routes

Why use it.

No configuration file.
Maintainable. Intuitive.
Simple.

Imagine multiple developers commiting code to the same branch, doing multiple stuff without conflicting.

POST /article/new/ - creates a new article
				GET /article/:id/ - get all data on article with given id
				PUT /article/:id/ - alter data on article with given id
				DELETE /article/:id/ - delete article with given id
-| routes
 | -| article
 |  | -| new
 |  |  | -| post.js
 |  | -| @id
 |  |  | -| get.js
 |  |  | -| put.js
 |  |  | -| delete.js
