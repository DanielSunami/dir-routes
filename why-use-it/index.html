dir-routes

Why use it.

There are many known methods to route your application, the most used ones are: mapping json files, writing one js file with all your routes and mount express routers on express routers. All these approuches involves writing more than one route to a single file and it can get a little bit confunsing on big projects. With dir-routes you have at least one file for each endpoint and that makes it possible to multiple developers commit code to the same branch without conflicting.

POST /article/new/ - creates a new article
GET /article/:id/ - get all data on article with given id
PUT /article/:id/ - alter data on article with given id
DELETE /article/:id/ - delete article with given id
