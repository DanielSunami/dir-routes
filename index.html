dir-routes

No configuration file.
Maintainable. Intuitive.
Simple. First to adopt this approach to routes.

Use your folder structure as routes, it uses the path as route and filename as HTTP method. Consider the following project folder:

-| node_modules
-| routes
 | -| new
 |  | -| @id
 |  |  | -| post.js
 | -| get.js
-| app.js

Inside the routes folder we have two files: /new/@id/post.js and /get.js; now lets take a look at each one of them.

/get.js

module.exports = function(req,res){
	res.send('hello world!');
}

Finally lets see the application entry point, app.js:

var express = require('express'),
	http = require('http'),
	app = express(),
	router = require('dir-routes');

app.use(router);

app.use(function(req, res) {
	res.status(404).end('404: Page not Found');
});

var httpServer = http.createServer(app);

httpServer.listen(8080, function() {
	console.log('Listening on port %d', httpServer.address().port);
});

npm install dir-routes
